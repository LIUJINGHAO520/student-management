## 🎓 设计原则

### 1. **视觉层次**
```
标题 (24px, 600w)
  ↓
副标题 (14-18px, 500w)
  ↓
正文 (14px, 400w)
  ↓
辅助 (12px, 400w, 灰色)
```

### 2. **颜色使用**
```
- 背景: 渐变色 (专业)
- 主要: 紫蓝渐变 (品牌色)
- 状态: 绿=好, 蓝=中, 红=差
- 中立: 灰色系
```

### 3. **交互反馈**
```
按钮悬浮: 向上 + 阴影加深
输入焦点: 边框变蓝 + 光晕
表格悬浮: 行背景变淡
删除按钮: 完全反转颜色
```

### 4. **动画**
```
进入: 0.3-0.4s, ease-out
过渡: 0.3s, ease
缓动: cubic-bezier 或 ease
```

## 🔍 常见问题

### Q: 为什么移除了 Bootstrap Grid？
A: Bootstrap Grid 基于 12 列系统，在现代设计中不如 CSS Grid/Flexbox 灵活。新方案性能更好，代码更清晰。

### Q: 在小屏幕上表格看不全怎么办？
A: 故意隐藏了非关键列（课程、教师、学期）。在大屏幕上会全部显示。这是移动优先的设计策略。

### Q: 为什么用 Emoji 作为图标？
A: 原生 Emoji 不需要加载资源，性能最优。如需自定义图标，可使用 Font Awesome 或 SVG。

### Q: 动画会影响性能吗？
A: 不会。所有动画都使用 `transform` 和 `opacity`，由 GPU 加速，不会触发 layout。

### Q: 能否改变颜色方案？
A: 可以。所有颜色都在 CSS 的最上面，易于修改。建议使用 CSS 变量进一步优化。
